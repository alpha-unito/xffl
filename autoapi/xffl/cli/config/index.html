xffl.cli.config¶

Guided configuration files creation for xFFL.

This script guides the user in the creation of the StreamFlow and CWL configuration files necessary to run xFFL workloads across different HPCs.

Attributes¶

logger

Default xFFL logger

Functions¶

_get_training_info(→ Tuple[xffl.custom.types.FileLike, ...)

Prompt user for training script path, parser path and parser name.

_get_model_info(→ Tuple[xffl.custom.types.PathLike, str])

Prompt user for model path and new model name.

_configure_facility(→ None)

Configure a single facility interactively.

_write_output_files(→ None)

Write StreamFlow, CWL and config files to disk.

config(→ int)

Run guided configuration for xFFL project.

main(→ int)

xFFL project's guided configuration entrypoint.

xffl.cli.config.config(args: argparse.Namespace) → int¶

Run guided configuration for xFFL project.

The user parser is imported twice on purpose:

  • once before facility loop to populate general training inputs (arg_to_bidding),

  • once per facility inside _configure_facility to re-import/re-evaluate arguments (this mirrors the original behaviour you requested).
